# Contract：外部事件源分类与分发

## 目的

约束命令、通知、退出三类事件源的允许动作、优先级与失败语义，避免外部路径直接改写领域状态。

## 事件分类

### 命令事件源

- 输入示例：CLI、管理口、远程控制输入
- 允许动作：解析为正式命令或触发请求，提交到受控入队路径
- 禁止动作：直接推进会话状态机、执行状态机或最近结果投影

### 通知事件源

- 输入示例：采集线程数据就绪、外设故障通知、缓存刷新完成
- 允许动作：更新最新快照缓存、登记通知、写 `eventfd` 唤醒调度器
- 禁止动作：直接改写会话状态、执行状态、最终结果

### 退出事件源

- 输入示例：EOF、关闭请求、停止运行信号
- 允许动作：触发直接退出或有界排空策略
- 禁止动作：无限排空、静默忽略退出请求

## 优先级规则

1. 退出事件优先于新的稳态周期推进。
2. 高优先级命令事件可在入队后触发一次受控补跑。
3. 通知事件只能影响缓存和唤醒，不抢占为独立领域推进入口。
4. 同类事件重复到来时，顺序和结果必须可重复。

## 可验证结果

- 命令事件仍可驱动启动/停止/故障命令
- 通知事件不会直接引发领域状态变更
- 退出事件会进入明确的排空或停止路径
- 多事件同时到来时顺序稳定
