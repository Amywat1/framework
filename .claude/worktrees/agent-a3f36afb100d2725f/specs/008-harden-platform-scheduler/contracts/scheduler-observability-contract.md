# Contract：调度器观测与只读状态视图

## 目的

约束调度器指标、最近错误原因和只读状态视图的最小对外语义，确保观测能力完整且不会反向改变领域状态或调度状态。

## 参与者

- `controller_scheduler`：提供只读观测接口
- Linux 私有实现：采集并维护指标
- 状态查询用例：只读投影调度器观测结果
- 日志端口：消费调度器节拍与事件观测输出

## 契约规则

1. 调度器必须至少提供周期次数、超拍次数、连续超拍次数、命令事件次数、通知事件次数、退出事件次数、挂起触发数量和最近错误原因。
2. 只读状态视图只能暴露观测结果，不得返回可变内部句柄、`fd` 或平台私有事件结构。
3. 观测读取不得推进领域状态、清空触发队列、改变运行状态或覆盖最近错误原因。
4. 超拍计数必须能区分累计超拍和连续超拍，并在非超拍周期正确清零连续计数。
5. 在异常与超拍验证场景下，观测输出必须稳定包含全部必需指标与最近错误原因。

## 可验证结果

- 状态查询能够读到完整调度器指标
- 日志输出能够区分不同事件源触发计数
- 最近错误原因在失败后可稳定读取
- 只读查询不会改变领域状态或调度器状态

## 失败语义

- 若观测视图无法生成，调度器必须返回明确错误，不得静默省略关键字段
- 若最近错误原因不存在，必须返回“无错误”语义，而不是未定义内容
