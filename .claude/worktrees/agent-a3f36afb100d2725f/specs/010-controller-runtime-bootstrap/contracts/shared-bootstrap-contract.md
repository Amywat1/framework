# 契约：正式入口与测试入口共用装配路径

## 目标

确保 `main.c` 与 `tests/test_support.h` 不再各自复制主控装配逻辑，而是通过正式 runtime/bootstrap 入口共享同一套闭环。

## 共享闭环

- 获取 `system_context`
- 绑定传感器与执行器端口
- 初始化程序仓储
- 初始化事件日志
- 创建 scheduler
- 运行前进入“已创建未运行”
- destroy 逆序回收

## 允许差异

- 驱动上下文实现
- 配置路径
- 日志路径
- stdio 绑定
- scheduler 参数
- 测试桩或辅助钩子

## 契约要求

- 测试辅助只能通过配置注入来替换依赖，不能复制正式装配步骤。
- 正式入口与测试入口的成功/失败语义必须一致。
- 正式入口与测试入口都必须遵守“单正式实例”“`run` 返回后终态”“正常退出算成功”这三条生命周期语义。
- 测试可以添加观测与注入能力，但不能绕开正式 lifecycle API。
