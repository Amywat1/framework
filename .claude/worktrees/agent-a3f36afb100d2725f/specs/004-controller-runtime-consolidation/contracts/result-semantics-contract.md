# 契约：结果记录与响应语义一致性

## 1. 目的

本契约定义最近结果、状态摘要、日志和 CLI 响应如何对齐到同一事件语义。

## 2. 统一事件结论

系统必须对以下事件类型形成统一结论：

- accepted
- rejected
- ignored
- error
- status

每个事件至少有：
- `result_code`
- `reason_code`
- 对应日志类型
- CLI `accepted=true|false` 表述

## 3. 最近结果刷新规则

以下事件都会刷新 `last_result_code` 和 `last_reason_code`：

- 命令被接受
- 命令被拒绝
- 命令被忽略

最近结果表示“最近一次被系统处理的事件结论”，而不是“最近一次真正引起状态变化的事件”。

## 4. 命令场景语义边界

- 本契约不在本轮新增或重定义任何单条命令的外部结果语义。
- `fault clear` 在有无 `global_fault` 两种场景下，均继续沿用当前既有产品语义。
- 本契约只要求同一事件一旦得出既有结论，CLI、状态摘要和日志必须共享该结论。

## 5. 对外表现一致性

- CLI 响应必须反映统一结果结论。
- `status` 必须能观察到与最近事件相一致的最近结果/原因语义。
- 日志不得和 CLI / 状态摘要对同一事件给出冲突结论。

## 6. 验收信号

- 接受、拒绝、忽略、迁移和故障清除场景中，最近结果、状态摘要、日志和 CLI 响应语义一致。
- `fault clear` 在相关场景中的既有语义经回归测试稳定保持，不因路径收敛发生漂移。
