# 契约：程序配置适配层

## 目标

定义外部 JSON 配置到内部强类型工步模型的唯一映射边界，确保字符串只存在于 `adapters/config`。

## 输入输出契约

| 项目 | 输入 | 输出 | 约束 |
|------|------|------|------|
| 程序加载 | JSON 文件路径 | `wash_program_v1_t` | 解析失败必须返回明确错误，不允许回退成伪成功内建程序 |
| 字段映射 | `"forward"`, `"tail"`, `"follow"` 等文本 | 方向、目标、控制模式等枚举 | 所有字符串在适配层一次性转完 |
| 合法性检查 | 字段组合 | 成功/失败结果 | 不支持的字段组合必须拒绝，不允许静默忽略 |
| 能力闭环 | RO 水/风干段配置 | 完整内部模型 | 若缺少对应执行机构能力必须报告缺口或在实现中补齐 |

## 外部 JSON 最小语义

| 配置块 | 必填语义 | 说明 |
|------|------|------|
| `program` | 程序标识、版本、启用状态 | 定义程序级元信息 |
| `segments[]` | 工步顺序、工步类型、段超时、收尾超时 | 定义完整工步链 |
| `motion_plan` | 方向、目标引用、目标值 | 定义工步移动目标 |
| `continuous_controls[]` | 控制对象、模式 | 定义持续控制 |
| `conditional_controls[]` | 启停条件、动作 | 定义按位置触发的控制 |
| `exit_actions[]` | 退出动作、完成反馈、超时 | 定义正式收尾 |
| `exception_policy` | 段内超时、收尾超时、异常策略 | 定义异常路径 |

## 映射约束

1. 领域层与应用层不得继续读取原始 JSON 文本。
2. 同一文本字段不得在多个层级重复解释。
3. 若配置声明了当前版本不支持的能力，必须返回失败或显式“不支持”，不得伪装为已生效。
4. 若 JSON 中仍使用旧阶段模型字段且与新工步段模型冲突，适配层必须按新模型拒绝或转换，不保留旧语义主导权。

## 验证准则

- 合法的工步段配置能完整映射为强类型模型。
- 非法字段组合、缺失退出动作、位置语义冲突、缺少执行机构品类映射的配置都会被明确拒绝。
- 不存在运行期继续通过字符串比较驱动核心逻辑的路径。
