# 契约：运行时结果投影

## 目标

定义领域事实到最近结果、状态摘要、日志和 CLI 响应之间的统一映射规则，避免对同一事件出现多套相互冲突的表达。

## 事件映射契约

| 事件类型 | 领域/应用结论 | 是否刷新最近结果 | 日志通道 | CLI 响应来源 | 状态摘要要求 |
|------|------|------|------|------|------|
| 接受 | `accepted` | 是 | `transition` | 统一投影结果 | 与最近结果一致 |
| 拒绝 | `rejected` | 是 | `rejection` | 统一投影结果 | 与最近结果一致 |
| 忽略 | `ignored` | 是 | `ignored` | 统一投影结果 | 与最近结果一致 |
| 状态迁移 | `accepted` / `completed` / `aborted` 等 | 是 | `transition` | 统一投影结果 | 迁移摘要与最近结果一致 |
| 超时决议 | `waiting` / `accepted` / `aborted` 等 | 是 | `transition` | 统一投影结果 | 决议语义一致 |
| 故障清除 | 沿用既有外部语义 | 是 | `transition` 或既有通道 | 统一投影结果 | 与最近结果一致 |
| `status` 查询 | 只读状态视图 | 否 | `none` 或沿用现有只读策略 | 状态视图格式化结果 | 只呈现当前状态，不制造新结果 |

## 投影约束

1. 最近结果只允许统一主触发编排入口写入。
2. 日志记录、`last_transition_record` 和 CLI 响应必须共享同一个事件结论来源。
3. `status` 查询不得通过查询动作刷新最近结果或改变最近原因。
4. 领域服务不得直接决定日志通道或 CLI 文本。

## 验证准则

- 对接受、拒绝、忽略、状态迁移、超时决议、故障清除进行回归时，最近结果、日志和 CLI 表达一致。
- `status` 响应与状态摘要一致，但不会让最近结果发生变化。
